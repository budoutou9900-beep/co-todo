# 機能仕様

co-todoの画面・機能一覧。UIの実装は主に [js/app.js](../js/app.js) に集約されている。

## 画面構成（3タブ + First モード）

下部タブバーで3画面を切り替える。加えて「First」という朝の儀式的なオーバーレイモードがある。

### 今日タブ（today）

その日（`state.selectedDate`）のタイムラインを表示する画面。通常は「今日」の日付を指すが、
今週タブの月カレンダーで別日をタップすると、その日の詳細表示としても使われる（後述）。

- **タイムライン表示**（[js/timeline.js](../js/timeline.js)）
  - Googleカレンダーの予定（終日予定は最上部固定、時刻ありは時刻順）とタスクが1本のリストに
    混在表示される。予定とタスクの間にタスクを差し込むように並び替えできる。
  - タスクは「今日中」セクションと「**+α**」セクションの2つに分かれる（`priority` フィールド）。
    +αは0件でも常に見出しが表示され、空でもドラッグでタスクを移動できる。
  - 完了済みタスクは折りたたみ可能な「完了済みタスク」セクションにまとめられる。
- **タスクの並び替え**: 長押し（タッチ）またはドラッグ（マウス）で並び替え。セクション（今日中⇔+α）を
  またいで移動すると `priority` も自動更新される（[js/drag.js](../js/drag.js)）。
- **左スワイプで削除**: タスク行を左にスワイプすると削除ボタンが出る（[js/swipe.js](../js/swipe.js)）。
- **タップで完了トグル / タップで編集**: チェック部分のタップは完了トグル、行全体のタップは
  編集シートを開く。
- **カレンダー連携トグル**（`📅 連携` チップ）: Googleカレンダーの読み取り専用連携をON/OFF。
- **Firstボタン**: 朝の儀式モード（後述）を起動。
- **日別詳細表示からの「戻る」ボタン**: 今週タブの月カレンダーから別日を閲覧しているときのみ表示され、
  タップで今日・今週タブに戻る。

### 今週タブ（week）

月カレンダー＋週内のタスク/予定リストを表示する画面。

- **月カレンダー**（[js/calendar.js](../js/calendar.js) の `renderMonthCalendar`）: 日曜始まり6週グリッド。
  各セルにGoogle予定とタスクのチップを最大3件、あふれた分は `+N` 表示。
  - **セルタップでその日の詳細表示に遷移**: 今日タブへ切り替わり、その日の予定＋タスクのタイムラインを
    見られる（従来はタスク追加が開いていたが、日付タップ＝閲覧という一般的なカレンダーアプリの
    挙動に変更済み）。タスク追加は今日タブ下部の＋ボタンから行う。
  - 月移動は左右の矢印（‹ ›）。
- **週内タスクリスト**（`renderWeekView`）: 暦週（月〜日等）ではなく「今日から7日間」のローリング
  表示。各日ごとに未完了タスクとGoogleカレンダー予定をグループ表示する。タスクをタップすると
  編集シートが開く（カレンダー予定は読み取り専用）。
  - **予定とタスクの見分け**: Googleカレンダー予定の行はカレンダーアイコン＋薄い背景トーン
    （`.week-event-row`）で表示され、丸ドットのタスク行と視覚的に区別される。

### プロジェクトタブ（projects）

プロジェクト（タスクのグルーピング単位）の一覧・管理画面。

- **長期/短期の自動グループ分け**: 締切日から自動判定してグループ表示する（保存フィールドは無く
  都度算出する派生値）。締切まで**15日以上先なら「長期」**、それ以外（**締切未設定を含む**）は
  「短期」。「短期」グループを先に、「長期」グループを後に表示する（`isLongTermProject`、
  [js/utils.js](../js/utils.js)）。
- **プロジェクトカード**: タイトル・進捗％（紐づくタスクの完了率、保存はせず都度算出）・
  完了数/総数・締切日を表示。締切超過は警告色になる。
- **開閉トグル**: カードをタップすると展開し、紐づくタスク一覧（サブタスク行）が表示される。
  開閉状態はFirestoreにも保存され、他デバイスでも同期される。
- **プロジェクト内タスクの並び替え**: 展開時、長押し/ドラッグでタスクの並び順を変更できる
  （今日タブと同じ `drag.js` を汎用利用）。並び順はFirestoreの `order` フィールド経由で
  他デバイスにも同期される。
- **プロジェクト内タスクの追加**: 展開時の「＋ タスクを追加」から、そのプロジェクトを
  プリセットした状態でタスク追加シートを開く。
- **左スワイプ＋確認モーダルで削除**: プロジェクトを左スワイプして削除ボタンをタップすると、
  確認モーダル（「削除する」「キャンセル」）が挟まる。誤って削除しないための二段階確認。
  削除すると紐づくタスクもまとめて削除される。
- **プロジェクト追加**: プロジェクトタブ右上の＋ボタンでアプリ内モーダル（タイトル・締切日の
  入力フォーム）が開く。デスクトップ版（Electron）で `window.prompt()` が動作しない問題への
  対応も兼ねて、ネイティブダイアログではなくアプリ内UIで実装している。色は12色のパレットから
  自動で順番に割り当てられる
  （手動選択UIは無い）。

### タスク編集シート（ボトムシート）

画面下からせり上がるシートで、タスクの追加・編集を行う（`renderSheet` / `openSheet`）。

- **タイトル**: テキスト入力。Enterキーで保存し、新規追加時は続けて次のタスクを入力できる
  （タイトルだけクリアして他フィールドは維持）。
- **日付**: ネイティブの `<input type="date">`。
- **プロジェクト**: タップで一覧ピッカー（ボトムシート）が開き、プロジェクト名＋色ドットの
  リストから1つ選ぶ（「なし」も選択肢に含む）。日付ピッカーと同様の直感的なUI。
- **優先度（今日中 / +α）**: 2択トグル。
- **繰り返し**: タップで巡回選択（なし → 毎週金曜日 → 3日ごと → 毎日）。
  完了時に次回分のタスクが自動生成される（[js/tasks.js](../js/tasks.js)）。

### First（朝の儀式モード）

「今日始めるタスクを選ぶ」ための簡易フローで、画面全体を覆うオーバーレイ。

- 候補は「未完了 かつ（日付が今日 or 日付未設定）」のタスクから最大3件選択。
- 「今日スタート」をタップすると、選んだタスクの `date` を今日に更新し、今日タブへ遷移する。

## 横断的な挙動

- **期限切れタスクの自動繰り上げ**（`carryOverOverdueTasks`）: 未完了・繰り返しなしのタスクで
  `date` が今日より過去のものは、Firestore購読の更新のたびに自動で今日の日付に書き換わる。
- **日付またぎへの追従**（`syncSelectedDateToToday`）: アプリを開きっぱなしで日付が変わった場合、
  1分ごと・画面復帰時に検知して `selectedDate` を新しい今日に更新する。ただし今週タブから
  別日の詳細を能動的に閲覧中は、この自動追従で勝手に今日へ戻されないようガードされている。
- **トースト通知**（`flash()`）: 画面下部に一時的なメッセージを表示（完了・削除・エラーなど）。
- **同一HTML再描画スキップ**: Firestoreの `onSnapshot` は同一データでも複数回発火することがあるため、
  直前と同じHTML文字列なら再描画をスキップし、CSSアニメーション（完了チェックのアニメ等）が
  途中で壊れないようにしている。

## Googleカレンダー連携の詳細

[js/calendar-sync.js](../js/calendar-sync.js) 参照。Firebase Authとは別に、Google Identity
Services (GIS) のトークンクライアントで読み取り専用スコープ（`calendar.readonly`）の
アクセストークンを取得する。

- ユーザーが閲覧可能な全カレンダー（購読カレンダー含む）を横断して予定を取得・統合する。
- **カレンダーごとの色**: `calendarList` APIの `backgroundColor` を使い、予定をどのカレンダーに
  由来するかで色分けする（Googleカレンダー本体の色分けに準拠）。取得できない場合は固定の
  フォールバック色を使う。
- 今日タブ・今週タブ双方でこの色分けが反映される。
- 連携状態は `localStorage` に保持し、次回起動時も維持される。

## デスクトップ版（Electronラッパー）

[desktop/](../desktop/) 参照。Chrome PWAのウィンドウ最小幅制約（約500px）を回避するために、
Electronで細い縦長ウィンドウ（既定380px）として本番URLをそのまま読み込む別配布。
ログインは埋め込みブラウザのOAuthがブロックされるため、システム既定ブラウザで認証し、
取得したトークンを `window.desktopAuth.googleOAuth()` 経由でWebアプリ側に渡す。
